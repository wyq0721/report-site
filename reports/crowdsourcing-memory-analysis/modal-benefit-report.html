收益性质(评委关注点)

① 削峰 > 降均值。 仓颉 live 堆在同场景内剧烈摆动(下线 63→140MB);峰值由"已死待 GC 的 JSON 瞬时树"撑起。modal 在作用域退出即时释放,把峰压向基线——直接降低 HarmonyOS 低内存查杀(LMK)风险。mobile 看峰值不看均值。
② 降 GC 开销。 这 ~30–46MB 瞬时不再进 GC 堆,非分代 copying collector 就不必逐字节 evacuate 它 → GC 更少更便宜(CPU/停顿收益)。
③ 堵泄漏面。 本 dump 取自 memory_leak 可靠性路径;JsonParser 钉树、解析树无 retainer 正是"瞬时被意外存活"现场,modal 从语言层消除该隐患。

口径与边界

  • 仓颉堆在 --mem 里的归属: 是命名匿名映射 [anon:cangjie_heap](源码 RegionSpace.cpp:92),计入 AnonPage other,不在 native heap(那是 malloc 家族)。占全应用 PSS ~8–10%(下线 711MB / 上线 889MB 总 PSS)。
  • live ≠ 物理足迹: cjprof 测存活对象字节;已 commit 的 region(含 copying 的 from/to 预留、碎片)更大。"可优化 MB"是逻辑对象量。
  • 判定依据: 类型聚合(cjprof 类型表)+ 逐类型逃逸抽查(--show-reference --incoming-reference 的 [ROOT GLOBAL] 标注);非逐实例全量逃逸分析。C3/C4 系数 0.3–0.6 为工程估计。
  • 采样: 每场景 3 次 cjheap,取最小=基线、最大=峰值;--mem 与 cjheap 非同时采集,故不做配对。
