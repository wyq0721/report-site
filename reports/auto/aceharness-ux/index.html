ACEHarness · UX 评审 v2

演进方向 + 期望界面稿（含业界对标）

基于 rc.1 版本评审，并对标 OpenHands、Devin、LangGraph Studio、Dify、n8n、CrewAI 等同类 harness / agent 编排产品。改版已解决「Agent 标签墙」「首页空状态」；本文档聚焦下一轮高价值演进，把业界成熟模式直接落成可照做的界面稿，用于引导开发。

P0 本轮必做 P1 计划内 P2 打磨 业界借鉴来源 期望效果标注

1演进方向一览

前两项为本地评审发现，后四项为业界对标后新增 —— 业界护城河已从「能编排」转向看得见、调得动、上手快，这正是 ACEHarness 当前最弱处。

P0

可视化状态机图 + 运行回放 借鉴 · LangGraph Studio / OpenHands

ACEHarness 已有「状态机 / 阶段 / 跨阶段回退」概念，但界面看不到图。LangGraph Studio 渲染执行图、每个节点可查看状态、checkpoint 存档、「时间旅行」回看每步决策、可在节点前后人工介入。
要做：把「运行记录」从列表升级为可视化流程图 + 时间轴回放：节点可点开看输入/输出/状态，支持回退到任一 checkpoint、人工干预后继续。→ 见稿 §4
P0

「新建工作流」向导化 + 模板库开场 借鉴 · n8n / Dify / CrewAI

现状弹窗铺开 8+ 字段，且默认「阶段模式」≠ 推荐「状态机模式」。业界共识：从模板开始、别给空表单（CrewAI 靠模板+角色化把 onboarding 做到最好）。
要做：拆 3 步（选模式 → 描述需求 → 高级·折叠），默认项=推荐项；第一步提供模板库（issue→PR、代码审查、红蓝对抗…）。→ 见稿 §3
P1

统一运行工作台 + 多任务并行视图 借鉴 · OpenHands Canvas / Devin Desktop

OpenHands 把编辑器/终端/浏览器/预览放一屏，多 agent 各自隔离在 git worktree、一屏可见；Devin Desktop 主打 plan→delegate→review→ship 一个界面完成。
要做：运行状态/记录整合为一个运行工作台，左侧多 run 列表、右侧四联窗（流程图/日志/产物/预览），带人工审核检查点。→ 见稿 §5
P1

Agent 角色化抽象 借鉴 · CrewAI

CrewAI 用 role / goal / backstory 建模，被评为最直观。ACEHarness 有 63 个 agent、阵营、指挥官，概念更丰富但缺统一心智。
要做：Agent 卡片统一为「角色 · 目标 · 擅长」三段式，配首次使用引导。
P1

信息架构精简 + 低代码/代码双模 借鉴 · n8n

侧边栏 ~14 项概念重叠（模型管理 vs 引擎管理）。且默认界面即「开发工程师界面」偏 dev；n8n 招牌是可视化节点 + 内联代码双模，兼顾两类人。
要做：侧边栏重排四组、合并模型+引擎；工作流保留 yaml（专家）+ 新增可视化画布（非技术），双向同步。→ 见稿 §6
P2

首页去重 + 按钮色彩语义 本地评审

首页 9+ 卡片与侧边栏重复、色彩过载；弹窗「取消」为红色（红=危险，误导）。
要做：首页保留「快速开始 3 卡 + 最近工作流」；取消改中性，蓝色只留主操作。→ 见稿 §7
