Agent Centric Engineering Harness

把 AI 工具升级为可治理的流程团队

ACEHarness 是仓颉团队出品的本地优先 AI Multi-Agent 流程协同平台。它把需求澄清、规则理解、任务拆解、状态机执行、Agent 分工、人工审核、长期记忆、Notebook 沉淀、引擎诊断和渠道接入放到同一个工作台里，让复杂业务与工程任务可规划、可执行、可审查、可回退、可复盘。

预算与滚动预测

用 Workflow 固化预测口径、历史数据拉取、差异解释和版本确认，Evidence 留下数据来源与人工确认。

月结与对账

用 State/Phase 管理关账阶段，用 Step 处理异常科目和跨系统差异，用人工门禁控制最终放行。

经营分析与报告

由分析 Agent 提取波动点，挑战 Agent 质疑解释链，报告 Agent 输出管理层摘要，Skill 提供口径和模板。

Capabilities

覆盖流程设计、执行调度、工具接入、证据治理和企业集成的完整运行能力。

从流程设计到执行治理，ACEHarness 提供六类支撑能力

Spec 与 Workflow 生成

把自然语言需求整理成目标、约束、任务拆分和工作流草案，既能服务预算分析，也能服务编译器修复和 SDK 迁移。

状态机与阶段执行

支持 state-machine 和 phase-based 两种模式，承载条件跳转、失败回退、检查点、人工审批、并行分组和子工作流。

Supervisor 与多 Agent 编排

根据角色、能力、上下文和引擎可用性调度 Agent，支持分析、执行、挑战、报告、裁决等职责分工。

Skill、MCP 与工具调用

把制度口径、RAG 检索、SQLite 工作缓存、构建测试、代码迁移、内部 API 等能力封装成 Agent 可调用的工具入口。

运行证据与审查面

运行记录、事件日志、步骤日志、审计日志、人工问题、Git baseline、step diff 和 Notebook 复盘共同形成证据链。

多引擎与渠道集成

通过 Engine、ACP、MCP、REST API、飞书、钉钉、微信 Bridge 和 Webhook，把 AI 执行与企业入口连接起来。

Integration

面向生态伙伴、内部平台和企业工具链的五类对接面。

可连接生态 Agent，也可作为编排后端

A

执行引擎接入

通过 ACP、SDK wrapper、HTTP adapter 或 CLI wrapper 接入第三方 Agent、模型服务或内部执行后端。统一 Engine 接口负责 execute、cancel、stream、availability 和上下文压缩。

B

MCP 工具供给

生态伙伴可以把检索、构建、部署、内部 API 或专有数据封装成 MCP Server，ACEHarness 内不同 Agent 可按任务调用。

C

渠道桥接

飞书、钉钉、微信 Bridge 和通用 Webhook 统一进入 Channel inbound API，把 IM 消息接入工作流和 Agent 对话上下文。

D

Skill 与插件分发

通过 Skill 包沉淀领域流程，通过侧边栏插件增加快捷动作、Tab 和意图，让财经、研发、运营等领域能力进入首页工作台。

E

REST API 嵌入

其他产品可以调用 runs、workflow、chat stream、models、channels、agora、schedules、spec-coding 等 API，把 ACEHarness 当多 Agent 编排后端。

生态协作方式灵活

已有 Agent 可以接入执行层；内部工具、制度文档、报表查询和专有数据可以封装为 MCP Server 或 Skill 包；已有前端和客户入口的产品也可以把 ACEHarness 作为本地优先的多 Agent 编排后端嵌入。

Differentiation

和常规 AI 聊天助手、AI IDE 的差异。

核心差异：以流程治理承载 AI 输出

维度 常规 AI 助手 ACEHarness
任务组织 以聊天、问答或单点工具调用为主，任务结构依赖用户手动维护。 以 Spec、状态机、阶段、审批、回退和运行记录组织任务。
Agent 协作 通常是单助手或简单角色提示。 支持 Supervisor 路由、挑战者校验、Judge 裁决、Agora 议场和一人公司办公室。
执行后端 绑定单一模型或单一厂商体验。 统一适配多引擎，支持 SDK、stdio、ACP、CLI 和原生 runtime 形态。
知识沉淀 上下文多停留在当前会话。 角色记忆、项目记忆、工作流经验、Notebook、RAG 和运行产物可长期复用。
治理能力 审计、准入、诊断和渠道桥接通常需要单独补齐。 内置用户体系、模型诊断、运行历史、渠道归属校验、配置管理和 API 文档。
Adoption

从场景试点到团队流程固化，路径清晰

D0

安装与初始化

安装 npm 包，运行 ace，创建管理员，选择语言、默认工作目录和可用 AI 引擎。验证本地服务、权限边界和 ACE_HOME 数据目录。

D1

接入模型与工具

在引擎管理页配置团队可用模型、内部工具、文档知识库或数据查询能力，运行诊断确认流式输出、结构化结果和多轮稳定性。

W1

固化首个工作流

选择一个预算分析、费用审核、报表生成、政策问答、经营分析或研发交付任务，固化为状态机，配置执行、挑战、裁决和人工审批节点。

M1

形成团队资产

沉淀 Agent 配置、Skill、Notebook、运行复盘、渠道桥接和标准接入清单，让新任务直接复用既有流程资产。

Enterprise Readiness

把 AI 试用推进到团队级使用时，治理能力可以同步接上。

面向企业落地的治理准备

执行引擎就绪检查通过模型/引擎诊断确认连接、流式输出、结构化结果、多轮稳定性和可用模型列表。
人工门禁与回退高价值业务或研发任务可以配置人工审批、挑战者校验、版本基线、运行恢复和步骤级结果审查。
渠道与密钥归属飞书、钉钉、微信、Generic Webhook、MCP 和引擎环境变量可纳入用户归属校验与脱敏日志。
