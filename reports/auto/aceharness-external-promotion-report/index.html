Agent Centric Engineering Harness

把 AI 工具升级为可治理的流程团队

ACEHarness 是仓颉团队出品的本地优先 AI Multi-Agent 流程协同平台。它把需求澄清、规则理解、任务拆解、状态机执行、Agent 分工、人工审核、长期记忆、Notebook 沉淀、引擎诊断和渠道接入放到同一个工作台里，让复杂业务与工程任务可规划、可执行、可审查、可回退、可复盘。

Positioning

把模型、工具、流程和知识组织成可持续运转的任务系统。

ACEHarness 是企业复杂任务的 Multi-Agent 编排层

超越单点代码补全和单轮问答

ACEHarness 不把价值限制在「一次对话给一个答案」。它关注的是完整任务闭环：一个需求如何被拆成 Spec，如何进入状态机，如何由多个 Agent 分工，如何通过挑战与裁决提升质量，如何留下可追溯过程，如何把经验沉淀到下一次任务。

Local-first Self-hosted Multi-Agent Spec Driven

对业务团队：让 AI 承担流程中的专业分工

预算、结算、经营分析、风控、审计、文档和沟通都可以拆成不同 Agent 角色，在统一流程中协同，而不是靠单个聊天窗口临时推进。

对平台团队：把分散 AI 能力纳入统一治理

团队可以把不同模型、内部系统、文档知识库、数据查询能力和审批渠道接入同一套流程，并通过诊断、审批、日志、运行记录和渠道桥接降低协作失控风险。

Finance Perspective

面向财经团队，重点不是“AI 会不会写代码”，而是“AI 能不能被纳入流程”。

把财经工作拆成可执行、可审核、可沉淀的 AI 流程

财经团队可以怎么理解 ACEHarness

ACEHarness 更像一个 AI 流程调度台：把预算编制、月结关账、报表解读、费用审核、经营分析、政策问答、异常归因等任务，拆成标准 Workflow，再把每一步交给合适的 Agent、工具和知识库执行。人仍然保留关键审核权，系统负责过程组织、证据沉淀和复用。

流程编排 角色分工 规则沉淀 人工门禁 证据留痕
Workflow
一类可复用业务流程，例如月度经营分析、费用合规审核、预算滚动预测、合同付款校验或管理报表生成。
Step
流程中的阶段动作，例如收集口径、拉取数据、识别异常、生成解释、交叉校验、输出报告、等待人工确认。
Agents
承担不同职责的 AI 角色，例如政策解释员、数据分析师、预算助手、审计挑战者、报告撰写员、复核裁决员。
Skills
可被 Agent 调用的领域方法和工具入口，例如报表口径查询、费用规则校验、同比环比分析、异常归因模板、文档生成规范。
Evidence
每次执行留下输入、判断依据、工具结果、人工审批、版本记录和复盘结论，形成后续流程优化和知识更新的依据。

预算与滚动预测

把预测口径、历史数据拉取、差异解释、风险提示和版本确认固化为流程，减少靠手工表格和口头经验推进。

月结与对账

把关账清单、异常科目、跨系统差异、责任人确认和复核记录纳入状态机，提升可追踪性和复用性。

经营分析与报告

由分析 Agent 提取波动点，挑战 Agent 质疑解释链，报告 Agent 生成面向管理层的结论和行动建议。

政策与规则问答

把制度、口径、FAQ 和历史判例沉淀为 Skill 与知识库，让回答带出处、可复核、可持续更新。

费用合规与审计

费用审核可以拆成规则匹配、异常识别、材料补充、风险评级和人工裁决，避免 AI 直接给结论而无过程。

跨团队协同

飞书、钉钉、微信或 Webhook 可把业务请求接入流程，状态、结论和待办回到原协同渠道。

Capabilities

用统一层级把复杂任务从“聊天”变成“流程”。

Workflow、Step、Agents、Skills 共同构成可执行流程

Workflow：定义业务闭环

把一类复杂任务固化为可复用流程，例如从需求接收、资料收集、规则判断、分析生成、人工确认到结果归档。

Step：拆出阶段动作

每个 Workflow 由 Step 组成，支持条件分支、人工审批、回退、子流程、并行组和运行恢复，避免流程只停留在提示词里。

Agents：承担专业分工

Supervisor 根据角色、能力、引擎可用性和上下文，把步骤分派给合适 Agent。一个任务可以组合分析者、执行者、审查者、报告者和裁决者。

Challenge & Judge：提高可信度

Defender 负责提出方案，Attacker 挑战边界和漏洞，Judge 做结构化裁决。高风险分析、合规判断和关键交付更适合这种模式。

Knowledge：沉淀规则与经验

角色记忆、项目记忆、工作流经验、会话补充记忆和 Notebook 协作空间一起工作，把制度、口径、复盘和历史案例变成可复用资产。

Skills：封装方法和工具

通过 Engine、ACP、MCP、REST API、Channel、Skill 和侧边栏插件扩展生态能力，让内部系统、数据查询、文档生成和领域流程都能被 Agent 调用。

Integration

面向生态伙伴、内部平台和企业工具链的五类对接面。

可连接生态 Agent，也可作为编排后端

A

执行引擎接入

通过 ACP、SDK wrapper、HTTP adapter 或 CLI wrapper 接入第三方 Code Agent。统一 Engine 接口负责 execute、cancel、stream、availability 和上下文压缩。

B

MCP 工具供给

生态伙伴可以把检索、构建、部署、内部 API 或专有数据封装成 MCP Server，ACEHarness 内不同 Agent 可按任务调用。

C

渠道桥接

飞书、钉钉、微信 Bridge 和通用 Webhook 统一进入 Channel inbound API，把 IM 消息接入工作流和 Agent 对话上下文。

D

Skill 与插件分发

通过 Skill 包沉淀领域流程，通过侧边栏插件增加快捷动作、Tab 和意图，让生态能力进入首页工作台。

E

REST API 嵌入

其他产品可以调用 runs、workflow、chat stream、models、channels、agora、schedules、spec-coding 等 API，把 ACEHarness 当多 Agent 编排后端。

生态协作方式灵活

已有代码 Agent 可以接入执行层；内部工具和数据可以封装为 MCP Server 或 Skill 包；已有前端和客户入口的产品也可以把 ACEHarness 作为本地优先的多 Agent 编排后端嵌入。

Differentiation

和常规 AI IDE/聊天助手的差异。

核心差异在工程治理，不在单轮回答

维度 常规代码助手 ACEHarness
任务组织 以聊天或 IDE 补全为主，任务结构依赖用户手动维护。 以 Spec、状态机、阶段、审批、回退和运行记录组织任务。
Agent 协作 通常是单助手或简单角色提示。 支持 Supervisor 路由、红蓝对抗、Judge 裁决、Agora 议场和一人公司办公室。
执行后端 绑定单一模型或单一厂商体验。 统一适配多引擎，支持 SDK、stdio、ACP、CLI 和原生 runtime 形态。
知识沉淀 上下文多停留在当前会话。 角色记忆、项目记忆、工作流经验、Notebook、RAG 和运行产物可长期复用。
治理能力 审计、准入、诊断和渠道桥接通常需要单独补齐。 内置用户体系、模型诊断、运行历史、渠道归属校验、配置管理和 API 文档。
Adoption

从单机试用开始，逐步沉淀为团队级标准流程。

从本地试用到团队流程固化，路径清晰

D0

安装与初始化

安装 npm 包，运行 ace，创建管理员，选择语言、默认工作目录和可用 AI 引擎。验证本地服务、CLI 和 ACE_HOME 数据目录。

D1

接入现有 Agent

在引擎管理页配置团队已有的 Claude Code、Codex、Cursor、OpenCode 或其他后端，运行模型/引擎诊断确认流式、结构化输出和多轮稳定性。

W1

固化首个工作流

选择一个编译器修复、SDK API、文档生成或性能分析任务，固化为状态机，配置 Defender、Attacker、Judge 和人工审批节点。

M1

形成团队资产

沉淀 Agent 配置、Skill、Notebook、运行复盘、渠道桥接和标准接入清单，让新项目复用既有流程，而不是重写提示词。

Enterprise Readiness

把试用体验推进到团队级使用时，治理能力可以同步接上。

面向企业落地的治理准备

执行引擎就绪检查通过模型/引擎诊断确认连接、流式输出、结构化结果、多轮稳定性和可用模型列表。
人工门禁与回退高价值研发任务可以配置人工审批、红蓝对抗、Git 基线、运行恢复和步骤级变更审查。
渠道与密钥归属飞书、钉钉、微信、Generic Webhook、MCP 和引擎环境变量可纳入用户归属校验与脱敏日志。
本地数据可控ACE_HOME 统一沉淀配置、缓存、日志、工作区、运行记录和 Notebook，便于私有化部署和运维管理。
